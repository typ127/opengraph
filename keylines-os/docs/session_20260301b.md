# Detaillierte Projekt-Chronik: OpenGraph / KeyLines OS

Diese Dokumentation listet chronologisch jede Interaktion, jeden Prompt und die daraus resultierenden technischen Änderungen auf.

---

## 24. Februar 2026: Die Geburtsstunde (Prototyping)

### Prompt 1: "hello"
*   **Aktion:** Initialisierung der Session und Begrüßung.

### Prompt 2: "i want to have a react based graph component showing nodes and edges..."
*   **Anforderung:** React-Komponente mit Force-Directed Layout (Spring) und Umschaltmöglichkeit auf hierarchisch (Top-Bottom / Left-Right). Datenhaltung in einer einfachen Graph-Datenbank. Referenz: KeyLines Basics.
*   **Aktionen:**
    *   Erstellung eines Vite-Projekts mit React und TypeScript.
    *   Installation von `cytoscape`, `react-cytoscapejs`, `cytoscape-dagre` und `cytoscape-fcose`.
    *   **Backend-Setup:** Erstellung eines Express-Servers (`server/index.js`) mit einer `db.json` als Mock-Datenbank.
    *   **Frontend-Setup:** Implementierung von `GraphComponent.tsx` (Cytoscape-Integration) und `App.tsx` (UI für Layout-Umschaltung).
    *   **Automatisierung:** Installation von `concurrently`, um Frontend und Backend mit einem Befehl (`npm run dev`) zu starten.
    *   **Bugfixing:** Behebung von TypeScript-Typisierungsfehlern in den Cytoscape-Erweiterungen.

### Prompt 3: "please check the alignment, the graph is correctly centered..."
*   **Anforderung:** Der Graph soll zentriert sein und der Container darf nicht unendlich wachsen.
*   **Aktionen:**
    *   Einführung der Klasse `.graph-wrapper` in `App.css` mit festen Größen (`height: 600px`, `max-width: 1000px`).
    *   Implementierung von `cy.fit()` und `cy.resize()` in `GraphComponent.tsx`, getriggert durch das `layoutstop`-Event, um eine perfekte Zentrierung nach jeder Animation zu garantieren.

---

## 26. Februar 2026: Der Pivot zu "KeyLines OS"

### Prompt 4: "/init"
*   **Aktion:** Analyse der neuen Projektstruktur (Umstellung auf FastAPI und Docker). Erstellung der initialen `GEMINI.md`.

### Prompt 5: "Act as a Senior Frontend Engineer... building an open-source alternative to KeyLines..."
*   **Anforderung:** Wechsel von Cytoscape zu **ReactFlow**. Custom Nodes mit MUI-Icons und SVG-Donut-Ringen für Prozentwerte. Knotenskalierung basierend auf einem 'Social Score' vom Backend. Implementierung der `/expand/{node_id}` Logik.
*   **Aktionen:**
    *   **Frontend-Umbau:** Erstellung von `KeyLinesNode.jsx` (MUI Icons + Donut-Logik).
    *   **State-Management:** Nutzung von `useNodesState` und `useEdgesState` in `App.jsx`.
    *   **Helper-Funktion:** Erstellung von `integrateNewData` für ein radiales Layout der neu geladenen Knoten.

### Prompt 6: "okay wie kann ich das nochmal ausführen?"
*   **Aktion:** Bereitstellung der Befehle für `docker-compose up` und den manuellen Start des Frontends.

### Prompt 7: "er findet keine index.html"
*   **Aktion:** Erstellung des `public`-Ordners und der `index.html`, da React-Scripts dieses Template zwingend benötigt.

### Prompt 8: "er sagt das bereits etwas auf port 3000 läuft"
*   **Aktion:** Erstellung einer `.env` Datei im Frontend, um die App auf **Port 3001** zu verschieben (da Memgraph Lab Port 3000 belegt).

### Prompt 9: "okay was sehe ich? Es gibt drei knoten... wo muss ich in memgraph nachehen?"
*   **Aktion:** Erklärung, dass aktuell noch Mock-Daten aus `main.py` kommen. Einführung in das **Memgraph Lab** und Bereitstellung der ersten Cypher-Abfrage (`MATCH (n) RETURN n`).

### Prompt 10: "ja. Vielleicht kannst du die mock daten behalten..."
*   **Anforderung:** Backend an Memgraph anbinden, aber Mock-Modus für Tests behalten.
*   **Aktionen:**
    *   Refactoring von `backend/main.py`: Integration von `gqlalchemy` zur Memgraph-Verbindung.
    *   Implementierung des `use_mock` Query-Parameters.
    *   Integration von `networkx` zur Berechnung der Knotengrößen (Centrality).

### Prompt 11: "ja" (Zustimmung zum Cypher-Skript)
*   **Aktion:** Erstellung eines umfangreichen Cypher-Skripts mit Testdaten (Hubs, Companies, Persons) inklusive Icons und Donut-Metadaten.

### Prompt 12: "im frontend sehe ich nur einen knoten, beim klich kommt {nodes: [], edges: []} zurück"
*   **Aktion:** Identifizierung des Problems: Die Query war zu restriktiv. Umstellung auf `OPTIONAL MATCH` in `main.py`, damit auch Knoten ohne Kanten zurückgegeben werden.

### Prompt 13: "leider immernoch alles leer. In der datenbank stehen n1 bis b6 in n.id"
*   **Aktion:** Tiefendiagnose. Erstellung von `debug_db.py`, um die Verbindung isoliert zu testen. Entdeckung von Syntaxfehlern im Diagnose-Skript und Fehlern bei der GQLAlchemy-Installation.

### Prompt 14: "backend-1 | Expanding node: n1... Found 1 rows..."
*   **Aktion:** Die "Heureka"-Minute. Das Log zeigte, dass Daten gefunden wurden, aber der Zugriff auf `.properties` fehlschlug.
*   **Lösung:** Implementierung eines extrem robusten Property-Extraktors in `main.py`, der nacheinander `.properties`, `._properties` und das Objekt selbst als Dictionary prüft.

## 27. Februar - 01. März 2026: Expansion & Polish (v1.23.0 - v1.27.0)

### Prompt 15: "ich bekomme aktuell den runtime error: ResizeObserver loop..."
*   **Aktion:** Behebung eines kritischen Fehlers in der React-MUI-Interaktion durch Stabilisierung der `useEffect`-Abhängigkeiten.

### Prompt 16: (Diverse Prompts zur Erweiterung der Funktionalität)
*   **Aktionen (Session 01. März):**
    *   **v1.23.0:** Implementierung von `localStorage` zur Speicherung visueller Präferenzen.
    *   **v1.24.0:** Erstellung des **Relationship-Editors**. Neuer Backend-Endpunkt `/update-edge` (Delete-and-Create Logik, da Cypher Typ-Änderungen nicht nativ unterstützt).
    *   **v1.25.0: Asimov-Datensatz.** Erstellung einer CSV-Import-Struktur für 40+ Knoten aus der Foundation-Serie. Einführung der Entität `book` und der `ExpandableText`-Komponente für Lore-Beschreibungen.
    *   **v1.26.0: Batch-Loading.** Implementierung von `Shift+Klick` in der Toolbox. Frontend sendet Anfrage an neuen `/nodes-by-type` Endpunkt. Anzeige der Objektanzahl in der Toolbox (z.B. "ROBOT (12)").
    *   **v1.27.0: UI-Finalisierung.** Implementierung des **Drill Down** (Isolation eines Knotens) und Fixes für Drawer-Scrolling-Probleme (feststehende Buttons).
